# Property Compare Brief

End-to-end feature for comparing multiple homes using hybrid property data and LLM-assisted scoring.

## Why this exists

Property buyers face scattered, conflicting, and incomplete information. This project focuses on:

- pulling property signals from multiple sources (hybrid real + mock)
- reconciling conflicting values into a confidence-aware brief
- letting users express personal needs and re-weight decision categories
- scoring each home with Claude (or deterministic fallback when keys are missing)

## Core user flow

1. Enter 2-5 addresses.
2. Express buyer priorities in natural language.
3. Adjust category weights (including groceries access).
4. Compare homes side-by-side with:
   - overall weighted score
   - per-category scores + explanations
   - conflict flags from reconciliation
   - blended core metrics (price, sqft, schools, walkability, groceries)

## Tech stack

- Next.js (App Router, TypeScript)
- Tailwind CSS
- Server route: `app/api/compare/route.ts`
- Claude API via `@anthropic-ai/sdk`
- Validation via `zod`

## Data strategy (hybrid)

- Mock multi-source records provide deterministic demo behavior.
- Placeholder adapters support real API wiring when keys are available:
  - `RENTCAST_API_KEY`
  - `WALKSCORE_API_KEY`
- Data is normalized and reconciled in `lib/reconciliation.ts`.

## Claude scoring strategy

- If `ANTHROPIC_API_KEY` is available, category scoring and explanations are generated by Claude.
- If not, a deterministic heuristic scorer is used so the app remains runnable and reviewable.

## Decision categories

- Neighborhood Quality
- Schools
- Commute & Transit
- Groceries Access

- Climate Risk
- Investment Potential

## Local setup

1. Install dependencies:
   ```bash
   npm install
   ```
2. Create env:
   ```bash
   # macOS/Linux
   cp .env.example .env.local
   # Windows PowerShell
   Copy-Item .env.example .env.local
   ```
3. Start:
   ```bash
   npm run dev
   ```
4. Open `http://localhost:3000`.

## Environment variables

See `.env.example`.

Minimum for AI scoring:

- `ANTHROPIC_API_KEY`

Optional for future real-source integrations:

- `RENTCAST_API_KEY`
- `WALKSCORE_API_KEY`

## Design notes and trade-offs

- **Hybrid source approach:** chosen for demo reliability without external API setup friction.
- **Reconciliation:** currently uses spread-aware averaging + conflict flagging for explainability and speed.
- **LLM risk control:** strict JSON response request plus fallback heuristic prevents hard failure.
- **Confidence model:** confidence is currently derived from source count and spread; not yet calibrated against labeled outcomes.
